# 🤖 FunsDiia Bot

Телеграм-бот для генерації та деплою персональних сторінок Дія.

---

## ⚙️ Налаштування GitHub Secrets

Перейдіть у ваше репо → **Settings → Secrets and variables → Actions → New repository secret**

| Secret | Опис | Приклад |
|--------|------|---------|
| `TELEGRAM_BOT_TOKEN` | Токен бота від @BotFather | `123456789:AAFxxxx` |
| `ADMIN_IDS` | ID адмінів через кому | `123456789,987654321` |
| `GROUP_CHAT_ID` | ID групи (від'ємне для супергрупи) | `-1001234567890` |
| `BOT_USERNAME` | Username бота без @ | `FunsDiia_bot` |
| `PAGES_GH_TOKEN` | GitHub Personal Access Token для деплою Pages | `ghp_xxxx...` |
| `REFERRAL_REWARD` | Реферальний бонус в гривнях (за замовчуванням 19) | `19` |
| `MIN_WITHDRAW` | Мінімальна сума виведення (за замовчуванням 50) | `50` |

---

## 🔑 Як отримати PAGES_GH_TOKEN

1. Відкрийте [github.com/settings/tokens](https://github.com/settings/tokens)
2. **Generate new token (classic)**
3. Дайте ім'я, наприклад `FunsDiia Bot`
4. Виберіть права: `repo` (повний) + `workflow`
5. Скопіюйте токен і додайте в Secrets як `PAGES_GH_TOKEN`

> ⚠️ Токен **ніколи** не вставляйте прямо в код — тільки через Secrets!

---

## 🚀 Як запустити

Бот запускається автоматично через GitHub Actions:
- При кожному push у гілку `main`
- За розкладом кожні 5 годин
- Вручну через вкладку **Actions → Run workflow**

---

## 📁 Структура папки `1/`

Папка `1/` — це шаблон сторінки, яка деплоїться на GitHub Pages.

- `values.js` — дані користувача (ПІБ, ДН, тощо) — **генерується автоматично**
- `1.png` / `sign.png` — фото паспорта — **замінюється на фото користувача**
- `index.html` — головна сторінка
- `assets/` — стилі, скрипти, зображення

> Всі файли з папки `1/` пушаться в новий репо при деплої.

---

## 📋 Workflow адміна

1. Користувач заповнює дані → бот надсилає замовлення **в групу і в чат адміна**
2. Адмін натискає **✅ Підтвердити оплату** → користувач отримує реквізити
3. Користувач надсилає чек → чек іде **в групу** з кнопкою **🚀 Деплой і надіслати**
4. Адмін натискає деплой → бот пушить файли на GitHub Pages, надсилає посилання **в групу** і користувачу
5. Адмін може надіслати посилання знову кнопкою **🔗 Надіслати посилання клієнту**
